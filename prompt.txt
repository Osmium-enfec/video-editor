You are a senior desktop-application engineer and video-processing expert.

Your task is to help me build a LOCAL DESKTOP VIDEO EDITING APPLICATION
(similar in spirit to Premiere/VLC but minimal and developer-focused).

--------------------------------
CORE CONSTRAINTS (VERY IMPORTANT)
--------------------------------
• This is a LOCAL desktop app (NOT web, NOT server-based).
• Videos are processed locally — NO uploads, NO APIs.
• Use FFmpeg as the core video/audio engine.
• Focus on performance, clean architecture, and lossless operations where possible.
• UI must NEVER freeze during FFmpeg execution.
• Keep things modular and extensible.

--------------------------------
TECH STACK (MANDATORY)
--------------------------------
• Language: Python 3.10+
• GUI: PySide6 (Qt)
• Video engine: FFmpeg CLI (invoked via subprocess)
• Preview: FFplay initially, upgradeable to QMediaPlayer
• Concurrency: threading / QThread (no async web frameworks)

--------------------------------
FEATURE SCOPE (PHASE 1)
--------------------------------
Implement the following features using FFmpeg:

1. Video Cut (lossless when possible)
2. Video Merge / Concatenate
3. Insert Black Screen (custom duration)
4. Blur Video
   - Full-frame blur
   - Region-based blur (x, y, width, height)
5. Audio Controls
   - Mute audio
   - Reduce volume
   - Fade in / fade out
6. Basic Noise Reduction (FFmpeg filters only)

--------------------------------

--------------------------------
Use the following project structure:

ffedit/

--------------------------------
• Use stream copy (-c copy) wherever possible
• Re-encode ONLY when filters are applied
• Allow user to choose quality presets when re-encoding
• Commands must be generated programmatically (NO hardcoding)
• Capture stderr to extract progress and errors
